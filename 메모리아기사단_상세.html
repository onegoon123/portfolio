프로젝트 개요

메모리아 기사단은 LLM API를 연동한 AI 기반 방치형 RPG 게임입니다. 플레이어는 영웅들을 수집하고 육성하며 AI 기반 대화 시스템을 통해 캐릭터들과 자연스러운 대화를 나눌 수 있습니다.

1. LLM API 연동 시스템

문제 상황

처음 구현했을 때 다음 문제들이 발생했습니다:

  • API 응답 시간이 3~5초로 길어 게임 프레임 저하
  • 요청이 많아질 경우 속도 제한(Rate Limit)으로 실패
  • API 비용이 예상보다 많이 발생

해결 과정

1단계: 비동기 처리 도입

기존 동기 방식으로 인한 프레임 저하를 해결하기 위해 비동기 처리를 도입했습니다.

개선된 방식

// 비동기 요청 구현
using (var cts = new CancellationTokenSource(timeout))
{
    string response = await LLM_API.SendMessageAsync(userInput, cts.Token);
    OnResponseReceived(response);
}

2단계: 요청 큐 및 디바운싱 구현

Rate Limit 문제를 해결하기 위해 요청 큐와 디바운싱을 구현했습니다.

public class AIRequestQueue
{
    private Queue<string> requestQueue = new Queue<string>();
    private bool isProcessing = false;
    private float lastRequestTime = 0f;
    private const float REQUEST_INTERVAL = 2.0f;

    public async void EnqueueRequest(string prompt)
    {
        requestQueue.Enqueue(prompt);
        if (!isProcessing)
        {
            await ProcessQueue();
        }
    }
}

3단계: 응답 캐싱

동일한 요청에 대한 반복 호출을 줄이기 위해 캐싱을 구현했습니다.

public class ResponseCache
{
    private Dictionary<string, CachedResponse> cache = new Dictionary<string, CachedResponse>();
    private const int CACHE_DURATION = 300; // 5분

    public bool TryGetResponse(string promptHash, out string response)
    {
        if (cache.TryGetValue(promptHash, out var cached))
        {
            if (Time.time - cached.timestamp < CACHE_DURATION)
            {
                response = cached.content;
                return true;
            }
        }
        response = null;
        return false;
    }
}

2. 뒤끝 백엔드 연동

문제 상황

데이터 로딩 시 다음 문제들이 발생했습니다:

  • 대량의 데이터를 한 번에 요청하여 로딩 시간이 길음 (5~8초)
  • 동기 요청으로 인한 게임 일시 정지 현상
  • 네트워크 오류 시 재시도 로직 부재

해결 과정

1단계: 데이터 분산 로딩

모든 데이터를 한 번에 로딩하지 않고 우선순위별로 분산 로딩했습니다.

  • 1단계: 필수 데이터 (로딩 화면)
  • 2단계: 백그라운드에서 로딩

2단계: 로컬 캐싱 구현

데이터를 로컬에 캐싱하여 로딩 시간을 단축했습니다.

public class DataCache
{
    public async Task<T> GetOrLoad<T>(string tableName, string inDate, string ownerInDate)
    {
        string cacheKey = $"{tableName}_{inDate}_{ownerInDate}";

        // 로컬 캐시 확인
        if (File.Exists(GetCachePath(cacheKey)))
        {
            T cachedData = LoadFromCache<T>(cacheKey);
            // 백그라운드에서 서버 동기화
            _ = SyncWithServer(cacheKey, tableName);
            return cachedData;
        }

        // 서버에서 로드 및 캐싱
        T serverData = await LoadFromServer<T>(tableName, inDate, ownerInDate);
        SaveToCache(cacheKey, serverData);
        return serverData;
    }
}

개발 도전과제 및 해결

비동기 프로그래밍

문제: C# async/await에 대한 이해도 부족으로 교착 상태(deadlock) 발생

해결: UniTask 라이브러리 도입, async Task 사용 규칙 정립, ConfigureAwait(false) 적용

API 보안

문제: API 키가 클라이언트 코드에 노출되는 보안 취약점

해결: 뒤끝 펑션(Function)을 프록시 서버로 활용, API 키는 뒤끝 서버에서만 관리

데이터 구조 설계

문제: 데이터 양이 증가함에 따라 유지보수가 어려워짐

해결: ScriptableObject 기반 데이터 구조로 직렬화, JsonUtility 활용, 에디터 확장으로 데이터 관리 툴 제작
