1. 객체 풀링 (Object Pooling)

문제 상황

전투 시 다음과 같은 성능 문제들이 발생했습니다:

  • 공격 효과가 100개 이상 생성되어 GC Spike 발생 (5~10초 지연)
  • 프레임이 30 FPS 미만으로 떨어짐
  • 메모리 사용량이 계속 증가

프로파일링 결과:

Profiler Sample:
- GC.Collect: 150ms (전투 중)
- AttackEffect.Instantiate: 80ms
- UI Element.Instantiate: 45ms

해결 과정

1단계: 기본 객체 풀 구현

재사용 가능한 오브젝트를 풀에 저장하고 필요할 때 꺼내 쓰는 방식을 구현했습니다.

public class ObjectPool<T> where T : Component
{
    private Queue<T> pool = new Queue<T>();
    private T prefab;
    private Transform parent;
    private int initialSize;

    public ObjectPool(T prefab, Transform parent, int initialSize)
    {
        this.prefab = prefab;
        this.parent = parent;
        this.initialSize = initialSize;
        PreWarm();
    }

    private void PreWarm()
    {
        for (int i = 0; i < initialSize; i++)
        {
            T obj = Object.Instantiate(prefab, parent);
            obj.gameObject.SetActive(false);
            pool.Enqueue(obj);
        }
    }

    public T Get()
    {
        if (pool.Count > 0)
        {
            T obj = pool.Dequeue();
            obj.gameObject.SetActive(true);
            return obj;
        }
        return Object.Instantiate(prefab, parent);
    }

    public void Return(T obj)
    {
        obj.gameObject.SetActive(false);
        pool.Enqueue(obj);
    }
}

3. ScriptableObject 기반 데이터 관리

문제 상황

데이터가 하드코딩되어 있어 다음 문제가 발생했습니다:

  • 수치를 변경할 때마다 코드 수정 필요
  • 데이터 추가 시 새로운 클래스 작성
  • 버전 관리 및 비교 어려움

해결 과정

1단계: ScriptableObject 데이터 구조

[CreateAssetMenu(fileName = "NewHero", menuName = "Data/Hero")]
public class HeroData : ScriptableObject
{
    [Header("기본 정보")]
    public string heroId;
    public string heroName;
    public Sprite icon;
    public GameObject prefab;

    [Header("능력치")]
    public int baseHP;
    public int baseAttack;
    public int baseDefense;

    [Header("성장률")]
    public float hpGrowthRate;
    public float attackGrowthRate;
    public float defenseGrowthRate;

    // 레벨별 능력치 계산
    public HeroStats GetStats(int level)
    {
        return new HeroStats
        {
            hp = Mathf.FloorToInt(baseHP * Mathf.Pow(hpGrowthRate, level - 1)),
            attack = Mathf.FloorToInt(baseAttack * Mathf.Pow(attackGrowthRate, level - 1)),
            defense = Mathf.FloorToInt(baseDefense * Mathf.Pow(defenseGrowthRate, level - 1))
        };
    }
}

개발 도전과제 및 해결

GC 최적화

문제: 방치형 게임의 특성상 지속적인 오브젝트 생성/삭제로 GC 발생

해결: Object Pooling 패턴 적용, 효과/UI/적 등 재사용 가능한 오브젝트 풀링, String Concatenation 대신 StringBuilder 사용

데이터 밸런싱

문제: 수백 개의 데이터 관리와 밸런싱의 어려움

해결: ScriptableObject로 데이터 직렬화, 에디터 확장으로 데이터 검증 툴 제작, 시각적인 데이터 편집기 구현

시나리오 연동

문제: Naninovel과 게임 로직의 통합 복잡성

해결: 커스텀 명령어로 게임 시스템 호출, ScriptableObject와 연동된 데이터 참조, 이벤트 기반 통신으로 느슨한 결합
