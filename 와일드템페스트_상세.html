1. GPGS 리더보드 시스템

문제 상황

기존 로컬 스코어 시스템은 다음 문제가 있었습니다:

  • 전역 랭킹 비교 불가
  • 친구와 경쟁 기능 부재
  • 기록 조작 가능성

해결 과정

1단계: GPGS 초기화

public class GPGSManager : Singleton<GPGSManager>
{
    private bool isAuthenticated = false;

    public void Initialize()
    {
        PlayGamesPlatform.Activate();
        AuthenticateUser();
    }

    private void AuthenticateUser()
    {
        Social.localUser.Authenticate(success =>
        {
            isAuthenticated = success;
            if (success)
            {
                Debug.Log("GPGS 인증 성공");
                LoadLeaderboard();
            }
        });
    }
}

3단계: UI 연동

실시간으로 랭킹을 표시하는 UI를 구현했습니다.

2. 모듈화된 아이템 시스템

문제 상황

아이템 시스템이 하드코딩되어 있어 다음 문제가 발생했습니다:

  • 새 아이템 추가 시 코드 수정 필요
  • 아이템 유형별 로직 분리 어려움
  • 확장성 부족으로 신규 아이템 추가에 오랜 시간 소요

해결 과정

1단계: 아이템 기본 클래스

[CreateAssetMenu(fileName = "New Item", menuName = "Data/Items/Item")]
public class ItemData : ScriptableObject
{
    [Header("기본 정보")]
    public string itemId;
    public string itemName;
    [TextArea] public string description;
    public Sprite icon;
    public GameObject prefab;

    [Header("분류")]
    public ItemType itemType;
    public ItemRarity rarity;

    [Header("효과")]
    public List<ItemEffect> effects;

    public virtual void Use(PlayerController player)
    {
        foreach (var effect in effects)
        {
            effect.Apply(player);
        }
    }
}

3. 뒤끝 실시간 데이터 동기화

문제 상황

데이터 동기화가 불안정했습니다:

  • 로그인/로그아웃 시 데이터 손실
  • 여러 기기에서 플레이 시 충돌
  • 네트워크 오류 복구 불가

해결 과정

1단계: 로봇 매니저 통합

뒤끝 서버와의 연결을 안정화했습니다.

2단계: 자동 동기화 시스템

public class AutoSyncManager : Singleton<AutoSyncManager>
{
    private float syncInterval = 30f; // 30초마다 동기화
    private float lastSyncTime = 0f;

    public async void StartSync()
    {
        if (isSyncing) return;
        isSyncing = true;

        try
        {
            await UploadChanges();
            await DownloadUpdates();
            lastSyncTime = Time.time;
        }
        catch (Exception e)
        {
            Debug.LogError($"동기화 실패: {e.Message}");
        }
        finally
        {
            isSyncing = false;
        }
    }
}

3단계: 충돌 해결 전략

  • 높은 스코어 유지
  • 골드 합산
  • 아이템 합집합

개발 도전과제 및 해결

GPGS 연동

문제: GPGS 인증 및 리더보드 설정의 복잡성

해결: Google Play Console에서 리더보드 ID 생성, 테스트 계정으로 미리 검증, 인증 실패 시 재시도 로직 구현

뱀서라이크 시스템 설계

문제: 무작위 생성과 난이도 밸런싱

해결: 절차적 생성으로 룸 배치, 난이도 곡선에 따른 적 강도 조절, 아이템 드롭 확률 조정

대규모 오브젝트 관리

문제: 동시에 많은 적과 효과가 화면에 표시

해결: Object Pooling으로 오브젝트 재사용, 비활성 오브젝트 비활성화, LOD (Level of Detail) 적용
